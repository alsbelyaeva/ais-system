# CRM–система для индивидуального специалиста
## Описание проекта

Данный проект представляет собой автоматизированную информационную систему, предназначенную для поддержки деятельности индивидуальных специалистов.  
Система позволяет вести учет клиентов, формировать заявки на занятия и автоматически оценивать удобство проведения занятий для преподавателя на основе настраиваемых весовых коэффициентов.


---

## Цель проекта

Целью проекта является разработка программного средства, автоматизирующего процесс:
- учета клиентов репетитора;
- формирования запросов на проведение занятий;
- расчета удобства запросов с учетом индивидуальных предпочтений преподавателя;
- отображения занятий в календаре.

---

## Основные функции системы

- регистрация и аутентификация пользователей;
- управление карточками клиентов;
- создание заявок на занятия с указанием времени;
- настройка весов критериев ранжирования;
- автоматический расчет удобства заявок;
- отображение подтвержденных занятий в календаре;
- разграничение прав доступа (администратор / преподаватель).

## Запуск проекта

### Требования
- Docker
- Docker Compose

### Инструкция по запуску

Проект контейнеризирован, что обеспечивает воспроизводимость среды выполнения. 
Для запуска проекта требуется установленный Docker и Docker Compose.

1. Клонировать репозиторий:

git clone https://github.com/alsbelyaeva/ais-system.git

2. Перейти в папку проекта:

cd ais-system

3. В корневой директории проекта выполните команду:

docker compose up --build

После запуска:

- Frontend доступен по адресу: http://localhost:5173
- Backend API доступен по адресу: http://localhost:4000

## Тестовые данные

В проекте предусмотрены тестовые данные (seed), позволяющие:

- войти в систему под ролями администратора и преподавателя;

- протестировать основные сценарии работы системы.

### Загрузка тестовых данных

После запуска контейнеров выполните команду:

docker compose exec backend npx prisma db seed

В базу данных будут добавлены:
- учетная запись администратора;
- учетные записи преподавателей;
- клиенты и примеры заявок на занятия.

### Отключение тестовых данных

Если тестовые данные не требуются, команду сидирования выполнять не нужно.
При этом система будет работать с пустой базой данных.
